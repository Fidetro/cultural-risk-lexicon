# 中文文化与历史风险关联库

使用本 skill 复核文本中可能存在的历史或文化风险关联。查询结果只能作为人工复核信号，不能当作最终审核结论。

## 工作流

1. 找出需要复核的词、名称、拼音、日期、编号或短语。
2. 在仓库根目录运行本地查询工具：

   ```bash
   python3 scripts/query.py "<token>" --json
   ```

3. 阅读每条结果的 `match_type`、`confidence`、`review_note` 和 `sources`。
4. 将结果解释为“可能关联”，不要在缺少上下文证据时推断创作者意图。
5. 如果命中来自模糊匹配或拼音别名，应说明关联强度弱于精确命中，并建议人工复核。

## 解释规则

- 精确别名命中强于数字错位、拼写近似或模糊匹配。
- 拼音别名的可靠性取决于数据条目本身，不能脱离来源和上下文扩展推断。
- 日期别名可能存在歧义，应结合周围文本是否构成历史语境。
- `731`、`0731`、`713` 等数字串都需要看上下文，尤其是编号、实验、部队、战争、侵华等语境。
- 优先使用“可能被读作”“存在关联”“建议人工复核”，不要直接写成“违规”。

## 数据文件

- `data/entities.json`：人物、军事部队、机构和相关标签。
- `data/events.json`：事件日期和纪念日期。

新增条目必须包含稳定来源链接。对别名、拼音和容易混淆的近似写法，应补充回归测试。
